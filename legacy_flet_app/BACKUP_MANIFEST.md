## Migration-Critical Modules (Priority Order)

1. **`db.py`** — All Supabase REST calls. Port to `supabase-dart` repository layer. ~850 lines of mechanical translation.
2. **`views/pricing.py`** — `on_save_and_generate()` is the most complex business logic (5-step save with rollback). Also contains `resize_product_image()` pipeline reference.
3. **`views/customer.py`** — `_get_category_items()` lazy-cache pattern, P4 tag filter, cart -> order flow.
4. **`views/settings.py`** — Tag Master CRUD with multi-category chips (complex UI logic).
5. **`views/orders.py`** — Order creation, mixed/single mode, share PDF flow.

## Warnings

1. **`card_path` column still exists** in Supabase `rate_list` table (all empty strings). Migration `sql/migration_remove_card_path.sql` not yet run. Defer or drop during Flutter migration.
2. **RLS is disabled** on `tag_master` table. Must re-enable with proper policies before Flutter production launch.
3. **Font licensing:** `HindiFont.ttf` is included but its license should be verified for redistribution in a published APK.
4. **APK builds are CI-only** — Windows local builds are broken due to Flutter 3.29.2 native-assets issue. Flutter migration will fix this.
